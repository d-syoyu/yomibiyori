# Typography Skill

## Purpose
This skill guides Claude to use more interesting and impactful typography choices in frontend design, avoiding generic fonts that create the "AI slop" aesthetic.

## Core Principles

### Avoid Generic Fonts
Never use these overused, boring fonts:
- Inter
- Roboto
- Open Sans
- Lato
- Default system fonts
- Arial

### Recommended Font Families

**Code Aesthetic:**
- JetBrains Mono
- Fira Code
- Space Grotesk

**Editorial:**
- Playfair Display
- Crimson Pro

**Technical:**
- IBM Plex family
- Source Sans 3

**Distinctive:**
- Bricolage Grotesque
- Newsreader

### Pairing Principle
High contrast = interesting typography
- Display + monospace
- Serif + geometric sans
- Variable font across weights

### Implementation Guidelines

**Use Extremes:**
- Weight: 100/200 vs 800/900, NOT 400 vs 600
- Size: Jumps of 3x or more, NOT 1.5x
- Create dramatic hierarchy

**Decision Making:**
- Pick ONE distinctive font
- Use it decisively throughout the design
- Load fonts from Google Fonts for reliability

**Strategic Application:**
Typography instantly signals quality. The right font choice elevates the entire design and communicates professionalism and attention to detail.

## Usage Instructions

When building frontends:
1. Select a distinctive font that matches the project context
2. Avoid defaulting to Inter or other generic choices
3. Create strong typographic hierarchy using extreme weight and size contrasts
4. Consider font pairing when using multiple typefaces
5. Load fonts properly from Google Fonts CDN
